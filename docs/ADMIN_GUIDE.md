# Guía administrativa

El administrador entra directamente por `/payment-organizer-finanzas/admin/` con el correo autorizado y su contraseña. No necesita registrarse en la página pública. La identidad permitida y el verificador de contraseña viven en Script Properties; el frontend no los conoce.

## Operación habitual

- Buscar un usuario y revisar plan, estado, sesiones y dispositivos.
- Asignar Premium con fechas y límite de dispositivos, indicando un motivo.
- Conceder o retirar funciones individuales del catálogo.
- Suspender, deshabilitar, revocar o reactivar, siempre después de confirmar.
- Administrar versiones, anuncios, configuración pública, contacto y errores.
- Consultar auditoría mediante el identificador de correlación.

No compartir tokens, hashes, códigos, notas internas ni identificadores técnicos completos. Ante pérdida de una sesión administrativa, revocar sesiones desde el backend y cambiar el verificador en Script Properties.
